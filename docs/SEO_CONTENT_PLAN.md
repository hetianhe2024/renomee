## 📁 目录结构说明

### 1. `file-types/` - 文件类型专题 ⭐
**定位**：针对不同文件格式的重命名需求  
**难度**：⭐ 低（容易做起来）  
**转化**：⭐⭐⭐ 中高

#### 子目录：
- `pdf/` - PDF 文件处理
- `images/` - 图片/照片处理
- `documents/` - Word/Excel 等办公文档
- `audio/` - 音频文件（MP3 等）
- `video/` - 视频文件

**典型关键词**：
```
rename pdf files by title
batch rename photos by date
organize jpg files
rename mp3 files by metadata
```

---

### 2. `use-cases/` - 使用场景专题 ⭐⭐⭐
**定位**：针对具体用户痛点场景  
**难度**：⭐⭐ 中（需要场景化内容）  
**转化**：⭐⭐⭐⭐⭐ 最高（意图明确）

#### 子目录：
- `downloads/` - 下载文件整理
- `research/` - 学术论文管理
- `business/` - 商务文档（发票、合同等）
- `media/` - 媒体文件管理（摄影、截图等）

**典型关键词**：
```
organize downloads folder
organize research papers
rename invoices automatically
organize screenshots
```

---

### 3. `how-to/` - 操作指南 ⭐⭐⭐⭐⭐
**定位**：直接回答"怎么做"的长尾关键词  
**难度**：⭐⭐ 中  
**转化**：⭐⭐⭐⭐ 高

**Google 最喜欢的类型**：明确的问题 + 步骤化解决方案

**典型关键词**：
```
how to rename pdf files by title
how to organize downloaded files
how to batch rename photos
how to rename files using AI
```

### 4. `alternatives/` - 竞品对比 ⭐⭐⭐⭐

## 📝 内容规范

### 长度要求
- **1000-2000 字**即可
- 重质量不重数量

### 必备元素
1. ✅ **明确的问题陈述**（用户痛点）
2. ✅ **3-5 步骤的解决方案**（Renomee 如何解决）
3. ✅ **截图/代码示例**（视觉化）
4. ✅ **指向主站的自然 CTA**（下载链接）
5. ✅ **相关指南的内链**（SEO 内链网络）

### SEO 优化
- **Title Tag**：包含目标关键词 + Renomee
- **Meta Description**：120-160 字，包含 CTA
- **URL**：简洁的 slug（如 `/how-to/rename-pdf-files/`）
- **H1**：与 Title 一致或语义相关
- **H2/H3**：包含长尾关键词变体
