# 常见问题

这里收集了用户最常问的问题和解决方案。

## 📥 安装与更新

### Q: 支持哪些操作系统？

**A:** Renomee 支持：

- ✅ Windows 10 (64位) 及更高版本
- ✅ Windows 11
- ✅ macOS 10.15 (Catalina) 及更高版本
- ✅ macOS 支持 Intel 和 Apple Silicon (M1/M2/M3) 芯片

暂不支持：

- ❌ Windows 7/8/8.1（已停止支持）
- ❌ 32位系统
- ❌ Linux（计划中）

### Q: 如何更新到最新版本？

**A:** Renomee 支持自动更新：

1. 有新版本时，右上角会显示更新通知
2. 点击通知查看更新内容
3. 点击 **"立即更新"** 自动下载并安装

也可以手动更新：

1. 访问 [GitHub Releases](https://github.com/hetianhe2024/renomee/releases)
2. 下载最新版本
3. 直接安装（会保留您的配置）

### Q: 更新会丢失我的配置吗？

**A:** 不会！更新时会自动保留：

- ⭐ 保存的预设
- ⚙️ 应用设置
- 📝 操作历史
- 📊 自定义元数据

建议：重要配置可以导出备份。

### Q: 安装后无法启动怎么办？

**A:** 请按以下步骤排查：

1. **检查系统要求** - 确保操作系统版本符合要求
2. **查看错误日志** - 日志位置：
   - Windows: `%APPDATA%\Renomee\logs\`
   - macOS: `~/Library/Application Support/Renomee/logs/`
3. **尝试重新安装** - 先完全卸载，再重新安装
4. **关闭杀毒软件** - 某些杀毒软件可能误报
5. **联系支持** - 发送日志文件到 support@xiaojingjia.top

## 🔧 功能使用

### Q: 如何批量重命名文件？

**A:** 三步搞定：

1. **添加文件** - 拖放文件夹到 Renomee 窗口
2. **输入规则** - 用自然语言描述，如 "添加前缀 Photo_"
3. **执行重命名** - 点击 "预览" 确认，然后 "执行重命名"

详见 [快速开始指南](quickstart.md)。

### Q: 支持哪些重命名规则？

**A:** Renomee 支持多种规则：

- 🗣️ **自然语言** - "把 IMG 替换成 Photo"
- 🔢 **变量系统** - `{date}_{n3}{ext}`
- 🤖 **AI 智能** - 自动识别文件内容
- 📝 **正则表达式** - `[REGEX] ^IMG_(\d+) → Photo_$1`
- ⚡ **JavaScript 脚本** - 自定义逻辑

详见 [基础使用](guide/basic-usage.md) 和 [高级功能](guide/advanced-features.md)。

### Q: AI 模式如何使用？

**A:** 启用 AI 模式的方法：

**方式 1:** 点击界面上的 **"🤖 AI模式"** 按钮

**方式 2:** 在规则前添加 `[AI]` 标记：

```
[AI] 按内容智能命名
```

AI 会自动分析：

- 📸 图片内容、拍摄信息
- 📄 文档标题、关键词
- 🎵 音乐标签信息
- 🎬 视频元数据

详见 [AI 智能模式](guide/basic-usage.md#ai-智能模式)。

### Q: 可以撤销重命名吗？

**A:** 可以！Renomee 提供两种撤销方式：

**方式 1：立即撤销**
- 重命名完成后，点击 **"撤销"** 按钮

**方式 2：历史记录撤销**
- 菜单 → 历史记录 → 选择操作 → 撤销

⚠️ **限制：**
- 只能撤销最近 10 次操作
- 如果原文件已被删除/移动，无法撤销

**建议：** 重要文件先备份！

### Q: 如何处理文件名冲突？

**A:** Renomee 提供三种冲突处理方式：

1. **自动编号**（推荐）
   ```
   Photo.jpg  →  Photo(1).jpg
   Photo.jpg  →  Photo(2).jpg
   ```

2. **跳过冲突**
   ```
   保持原文件名不变
   ```

3. **覆盖文件**
   ```
   ⚠️ 警告：会覆盖已存在的文件！
   ```

设置位置：设置 → 重命名 → 冲突处理

### Q: 可以只重命名特定类型的文件吗？

**A:** 可以！使用文件过滤功能：

1. 添加文件后，点击 **"过滤"** 按钮
2. 选择文件类型：
   ```
   ☑ 图片 (.jpg, .png, .gif)
   ☐ 文档 (.pdf, .docx, .txt)
   ☑ 视频 (.mp4, .avi, .mov)
   ```
3. 或自定义扩展名：`.jpg, .png`

详见 [文件过滤](guide/basic-usage.md#文件过滤)。

## 🛡️ 安全与隐私

### Q: Renomee 安全吗？

**A:** 完全安全！Renomee 承诺：

- ✅ **本地处理** - 所有文件在本地处理，不上传云端
- ✅ **无需联网** - 除 AI 模式外，完全离线工作
- ✅ **开源透明** - [源代码公开](https://github.com/hetianhe2024/renomee)
- ✅ **无广告** - 不包含任何广告或跟踪代码
- ✅ **隐私保护** - 不收集任何个人信息

查看 [隐私政策](about/privacy.md)。

### Q: AI 模式需要上传文件吗？

**A:** 分两种情况：

**智能识别模式（不上传）：**
- 图片 EXIF 数据提取 - ✅ 本地处理
- 文档元数据提取 - ✅ 本地处理
- 音视频标签读取 - ✅ 本地处理

**AI 内容分析模式（可选上传）：**
- 图片内容识别 - 需要上传缩略图（< 100KB）
- 文档关键词提取 - 只上传文本内容

**隐私保护：**
- 所有数据加密传输
- 不保存任何文件
- 可在设置中完全禁用

### Q: 重命名后原文件会丢失吗？

**A:** 不会！Renomee 只修改文件名，不修改文件内容。

**额外保护：**

1. **预览模式** - 执行前查看所有更改
2. **撤销功能** - 可以恢复原文件名
3. **操作日志** - 记录所有操作
4. **备份选项** - 可开启自动备份（设置 → 高级 → 备份原文件名）

### Q: 会修改文件的创建/修改时间吗？

**A:** 默认不会！Renomee 只修改文件名。

如果需要修改时间戳，可以：

1. 启用 **"保留时间戳"** 选项（设置 → 重命名）
2. 或使用变量 `{preserve:timestamp}`

## ⚙️ 技术问题

### Q: 为什么某些文件无法重命名？

**A:** 可能的原因：

1. **文件被占用**
   ```
   解决：关闭正在使用该文件的程序
   ```

2. **权限不足**
   ```
   解决：右键 Renomee → 以管理员身份运行
   ```

3. **文件名包含非法字符**
   ```
   非法字符: \ / : * ? " < > |
   解决：Renomee 会自动替换为下划线
   ```

4. **文件名过长**
   ```
   Windows 限制: 260 字符
   macOS 限制: 255 字符
   解决：使用更短的文件名
   ```

5. **文件位于只读目录**
   ```
   解决：更改文件夹权限
   ```

### Q: 处理大量文件很慢怎么办？

**A:** 优化性能的方法：

1. **关闭预览模式** - 直接执行重命名
2. **禁用 AI 模式** - 如果不需要 AI 识别
3. **使用简单规则** - 避免复杂的正则或脚本
4. **分批处理** - 每次处理 500-1000 个文件
5. **增加内存** - 设置 → 高级 → 最大内存使用

性能参考：

| 文件数量 | 预计耗时 |
|---------|---------|
| 100 个 | < 1 秒 |
| 1,000 个 | 2-5 秒 |
| 10,000 个 | 20-30 秒 |
| 100,000 个 | 3-5 分钟 |

### Q: 支持网络驱动器和 NAS 吗？

**A:** 支持，但需要注意：

1. **启用网络缓存** - 设置 → 高级 → 网络缓存
2. **增加超时时间** - 设置 → 高级 → 网络超时（秒）
3. **检查网络连接** - 确保网络稳定

**最佳实践：**

- 先复制文件到本地
- 在本地处理
- 处理完后再复制回网络驱动器

### Q: 命令行模式如何使用？

**A:** 基本语法：

```bash
renomee --rule "{date}_{n3}{ext}" --path "/photos"
```

常用选项：

```bash
--rule <规则>      # 指定重命名规则
--preset <名称>    # 使用预设
--path <路径>      # 文件或文件夹路径
--recursive        # 递归处理子文件夹
--dry-run          # 预览模式
--yes              # 自动确认
```

详见 [命令行模式](guide/advanced-features.md#命令行模式)。
