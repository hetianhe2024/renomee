本文档总结产品核心功能与差异化优势，为对外推广、PR传播、网站宣传提供参考

📌 产品定位

AI-Rename 是一款基于自然语言的智能文件管理工具，让用户通过对话完成文件批量操作，无需学习复杂命令。

核心价值主张：
- 🗣️ 自然语言操作 - 用对话代替命令行，告别复杂的批处理脚本
- 🧠 内容理解能力 - 不只看文件名，能读懂文件内部内容
- 🎯 智能意图识别 - 自动理解用户真实需求，无需精确表达
- ⚡ 即开即用 - Electron 桌面应用，每日 20 次免费试用

1.1 命名能力详解

AI 可以使用以下所有维度的信息来构造新文件名，用户只需用自然语言描述需求即可。

A. 文件系统元数据字段（无需提取，始终可用）

这些字段由客户端直接读取，不消耗提取权限，任何文件都可以使用：

字段 说明 命名示例
filename 完整文件名（含扩展名） report.pdf
file_basename 不含扩展名的文件名 report
file_extension 扩展名（含点号） .pdf
filesize 文件大小（字节） 用于"大于 1MB 的文件加 _large 后缀"
modification_time 最后修改时间（datetime 对象） 20240316_report.pdf
creation_time 文件创建时间（datetime 对象） 2024-03-16_report.pdf

日期格式化支持（modification_time / creation_time）：

格式代码 输出示例 适用场景
%Y%m%d 20240316 紧凑日期前缀
%Y-%m-%d 2024-03-16 标准日期
%Y年%m月%d日 2024年03月16日 中文日期
%Y%m%d_%H%M%S 20240316_143022 精确到秒
%Y / %m / %d 2024 / 03 / 16 单独年/月/日

B. 命名操作类型（用自然语言描述即可）

① 添加前缀

用户说法 效果示例
"在文件名前加上日期" report.pdf → 20240316_report.pdf
"前面加上 [已审核]" 合同.docx → [已审核]合同.docx
"加上序号前缀，从 001 开始" file.jpg → 001_file.jpg
"前面加上项目名 ProjectA" 需求.md → ProjectA_需求.md
"按修改时间排序后加序号" 时间最早的文件编号 001，依次递增

② 添加后缀

用户说法 效果示例
"在文件名后加上 _final" 报告.docx → 报告_final.docx
"加上修改日期后缀" 合同.pdf → 合同_20240316.pdf
"大于 10MB 的文件加 _large" video.mp4 → video_large.mp4
"加上版本号 _v2" 设计稿.psd → 设计稿_v2.psd

③ 删除/清理字符

用户说法 效果示例
"去掉文件名中的空格" my file.txt → my_file.txt
"删除括号里的内容" 报告(草稿).docx → 报告.docx
"去掉 _copy 后缀" 文件_copy.pdf → 文件.pdf
"删除文件名中的数字" report2024v3.pdf → reportv.pdf
"去掉特殊符号，只保留中文和字母" 【重要】report#1.pdf → 重要report1.pdf
"去掉 UnknownCompany 这段文字" 合同_UnknownCompany_20240101.pdf → 合同_20240101.pdf

④ 替换字符

用户说法 效果示例
"把空格替换成下划线" my file name.txt → my_file_name.txt
"把中文括号换成英文括号" 文件（草稿）.docx → 文件(草稿).docx
"把 - 替换成 _" file-name-2024.pdf → file_name_2024.pdf
"把 final 替换成 v2" report_final.pdf → report_v2.pdf

⑤ 大小写转换

用户说法 效果示例
"文件名全部改成小写" MyFile.TXT → myfile.txt
"文件名全部改成大写" report.pdf → REPORT.pdf
"首字母大写" my report.pdf → My report.pdf

⑥ 格式化与重组

用户说法 效果示例
"按 日期_原文件名 格式重命名" 报告.pdf → 20240316_报告.pdf
"按文件大小分类：大文件加 _HD，小文件加 _SD" 按大小自动分档
"按修改时间排序，加三位序号" 001_最早文件.pdf, 002_次早文件.pdf...
"把文件名中的年份统一改成 2024" report_2023.pdf → report_2024.pdf

⑦ 条件筛选后操作

用户说法 效果示例
"只重命名 JPG 文件" 其他格式保持不变
"只处理大于 5MB 的文件" 小文件跳过
"只重命名 2024 年之后修改的文件" 按时间筛选
"只处理文件名包含 draft 的文件" 精确匹配

2. 文件内容智能提取 🔥 核心差异化功能

2.1 支持的文件格式与可提取字段（完整规格）

提取方式说明：
- 🏠 本地提取：在用户电脑上完成，文件内容不上传服务器，隐私安全
- ☁️ OCR 上传：扫描版 PDF / 图片文字识别需上传服务器，需用户单独授权

📄 PDF 文件（.pdf）

提取方式：🏠 本地提取（文字版）/ ☁️ OCR（扫描版）
文件大小限制：≤ 50 MB
提取页数：最多前 10 页，文本预览截取前 1000 字符

可用字段 字段路径 说明
文档标题 metadata.title PDF 元数据中的标题，或从正文首行智能提取
作者 metadata.author PDF 元数据中的作者信息
主题 metadata.subject PDF 元数据中的主题
关键词 metadata.keywords PDF 元数据中的关键词
正文预览 text_preview / page_1 前 1000 字符的正文内容
章节标题 headings 自动识别的章节标题列表（最多 10 条）
总页数 page_count PDF 总页数
是否扫描版 is_scanned true 表示扫描版，需 OCR 才能识别文字

命名场景示例：
- "根据 PDF 标题重命名" → 使用 metadata.title
- "把论文按作者名整理" → 使用 metadata.author
- "根据文档第一段内容命名" → 使用 text_preview 前 50 字符
- "识别发票内容重命名" → 扫描版触发 OCR，提取金额/日期

📝 Word 文档（.docx）

提取方式：🏠 本地提取
文件大小限制：≤ 50 MB
提取范围：前 2 页文本 + 文档元数据

可用字段 字段路径 说明
文档标题 metadata.title Word 文档属性中的标题
正文预览 text_preview / page_1 文档正文前 1000 字符
章节标题 headings 自动识别的标题层级列表

命名场景示例：
- "用文档标题作为文件名" → 使用 metadata.title
- "根据合同第一段提取甲方名称" → AI 从 text_preview 中语义提取

📊 Excel / CSV（.xlsx .xls .csv）

提取方式：🏠 本地提取
文件大小限制：≤ 50 MB
提取范围：前 3 个工作表，每个工作表前 10 行

可用字段 字段路径 说明
工作表名称列表 sheet_names 所有工作表名称，如 ["销售", "库存", "汇总"]
列标题（第一行） column_headers 每个工作表的第一行内容
总行数 row_count 所有工作表的总行数
文本预览 text_preview 第一个工作表前 10 行的文本内容

命名场景示例：
- "按 Excel 第一个工作表名重命名" → 使用 sheet_names[0]
- "根据表格的列标题命名" → 使用 column_headers[0] 拼接
- "把月报文件按 Sheet 名整理到对应文件夹" → 按 sheet_names 分类

🖼️ 图片（.jpg .jpeg .png .heic .webp .tiff）

提取方式：🏠 本地提取（EXIF 元数据）/ ☁️ OCR（图片中的文字）
文件大小限制：无硬性限制（EXIF 提取极快）

可用字段 字段路径 说明
拍摄时间 exif.DateTimeOriginal / creation_date 相机快门按下的时刻
相机品牌 exif.Make 如 Apple, Canon, Sony
相机型号 exif.Model 如 iPhone 15 Pro, EOS R5
镜头型号 exif.LensModel 如 24-70mm f/2.8
光圈 exif.FNumber 如 f/2.8
快门速度 exif.ExposureTime 如 1/500s
ISO exif.ISO 如 ISO800
焦距 exif.FocalLength 如 50mm
GPS 坐标 gps.latitude / gps.longitude 拍摄地点经纬度
图片宽度 width / dimensions.width 像素宽度
图片高度 height / dimensions.height 像素高度
设备/软件 exif.HostComputer / exif.Software 如 iPhone 15 Pro

命名场景示例：
- "照片按拍摄日期重命名" → 使用 exif.DateTimeOriginal，格式 20240316_143022.jpg
- "按相机型号分类" → 使用 exif.Model 建立子文件夹
- "根据分辨率重命名，4K 的加 _4K 后缀" → 使用 width × height 判断
- "按拍摄参数命名" → 组合 f/2.8_1-500s_ISO800_50mm.jpg

🎵 音频（.mp3 .flac .wav .aac .ogg .m4a .wma）

提取方式：🏠 本地提取（ID3/音频标签）
文件大小限制：无硬性限制

可用字段 字段路径 说明
歌曲标题 audio_tags.title ID3 标签中的曲目名称
艺术家 audio_tags.artist 演唱者/艺术家
专辑艺术家 audio_tags.albumartist 专辑主艺术家（合辑时有用）
专辑名 audio_tags.album 所属专辑
年份 audio_tags.year 发行年份
流派 audio_tags.genre 音乐风格，如 Pop, Jazz
曲目编号 audio_tags.track 专辑中的曲目序号
作曲 audio_tags.composer 作曲家
作词 audio_tags.lyricist 作词人
制作人 audio_tags.producer 制作人
厂牌 audio_tags.label 唱片公司
语言 audio_tags.language 歌曲语言
时长 duration 秒数（浮点数），如 267.5
比特率 bitrate bps，如 320000（即 320kbps）
采样率 sampleRate Hz，如 44100
编码格式 codec 如 MP3, FLAC, AAC

命名场景示例：
- "按 艺术家 - 歌曲名 格式重命名" → 周杰伦 - 晴天.mp3
- "按专辑整理到文件夹" → 使用 audio_tags.album 建立子目录
- "加上曲目序号前缀" → 01_晴天.mp3
- "320kbps 的音乐加 _HQ 后缀" → 判断 bitrate >= 320000
- "按发行年份分类" → 使用 audio_tags.year 建立年份文件夹

🎬 视频（.mp4 .mkv .webm .avi .mov .m4v）

提取方式：🏠 本地提取（视频元数据）
文件大小限制：无硬性限制（只读取元数据，不加载视频流）

可用字段 字段路径 说明
视频标题 audio_tags.title 视频文件的标题标签
导演/指挥 audio_tags.conductor 导演信息（如有标签）
编剧 audio_tags.writer 编剧信息（如有标签）
年份 audio_tags.year 制作年份
视频宽度 width 像素宽度，如 1920
视频高度 height 像素高度，如 1080
帧率 fps 如 30, 60
时长 duration 秒数，如 5400（90分钟）
比特率 bitrate 总比特率（bps）
编码格式 codec 如 H.264, H.265, AV1

命名场景示例：
- "按分辨率重命名，1080p 的加 _1080p 后缀" → 判断 height == 1080
- "4K 视频和 1080p 视频分别放到不同文件夹" → 按 height 分类
- "加上时长后缀" → 视频_90min.mp4
- "按编码格式分类" → 使用 codec 建立子目录

📃 文本类文件（.txt .md .log .csv .json .xml .html 等）

提取方式：🏠 本地提取
文件大小限制：≤ 10 MB
支持格式：.txt .md .log .csv .json .xml .html .htm .css .js .ts .jsx .tsx .vue .py .java .c .cpp .go .rs .sh .bat .ps1

可用字段 字段路径 说明
文本预览 text_preview 前 500 字符的文本内容
文本总长度 text_length 字符数
Markdown 标题 headings # 开头的标题列表（最多 10 条，仅 .md）
CSV 列标题 column_headers 第一行内容（仅 .csv）
CSV 行数 row_count 总行数（仅 .csv）
JSON 顶层键 keywords 顶层 key 列表（最多 20 个，仅 .json）

编码支持：自动尝试 UTF-8 → GBK → Latin-1

命名场景示例：
- "根据 Markdown 文件的第一个标题重命名" → 使用 headings[0]
- "根据 JSON 文件的顶层键命名" → 使用 keywords 拼接
- "根据日志文件第一行内容命名" → 使用 text_preview 前 50 字符
- "CSV 文件按列数命名" → 使用 column_headers[0].length

2.3 内容感知代码生成（技术优势）

业界首创：两次 LLM 调用机制

传统方案：
  用户指令 → LLM 生成通用代码（盲写）→ 执行 → 结果不够智能

AI-Rename 方案：
  用户指令 → LLM 分析意图 → 请求提取权限 → 前端本地提取内容
           → 第二次 LLM 调用（看着实际内容生成代码）→ 精准命名
